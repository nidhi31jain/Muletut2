The file connector enables the mule application to read and write files. It can be implemented as an inbound endpoint or outbound one.

File connector supports the one-way exchange pattern only.

File Connector as an inbound endpoint

  • Place the file connector to the source section of flow.
  • Set the path to read the file from.
  • The file can also be moved to another location after reading by specifying Move to Directory option.
  • It checks for file in the set location after certain frequency called polling frequency.
  • It triggers the flow on receiving the file.
  • It reads the file into payload as a FileInputStream.
  • Regex filter can be used to specify the type of file to read or the file name itself.

By default, file connector deletes the file after reading.

File Connector as an outbound endpoint

  • It can create new files or append content to the existing ones.
  • Place the HTTP connector to source section of the flow and configure.
  • Place the file connector after HTTP.
  • Specify the path to write to and name of the file.
