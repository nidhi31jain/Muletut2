Mule can process message in batches. It has the ability split payload into a batch of records and process them, report them or just push to another system. This is particularly handy when processing a large file or a large set of data from database.

A batch job is an independent block of code with four phases.

  • Input: Have a message or source or series of processors to prepare data.
  • Load and Dispatch: Mule splits the payload into a collection and save them in queue. It is an implicit phase.
  • Process: Each records is processed asynchronously and moves through the processors.
  • Complete: Reports summary of records processed. This phase can be used to get some insight into the processed records.
A Batch Job can be triggered via
  • A Batch Execute element, to reference the Batch Job from a flow
  • Placing an inbound one way source like File or FTP in the input section of Batch Job
