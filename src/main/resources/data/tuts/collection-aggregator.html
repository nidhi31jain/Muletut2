The Collection Splitter, like its name, splits a collection and sends one fragment at a time to to the next message processor.
  • In the Collection Splitter flow, add a Collection Aggregator.
  • Drop a Logger after Collection Aggregator, write #[payload] in the message field.
  • The output in console by the last logger will be 
    					
[{id=1, name=Jane, surname=Doe}, {id=2, name=John, surname=Doe}]
    The order can be different.
