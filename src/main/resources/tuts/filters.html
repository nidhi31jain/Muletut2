In Mule, Filters are used to determine if a message will be processed or not. The message is passed to the filter and if condition is not met then the filter returns null value which in turn halts the processing the message.
Mule gives option of total 12 filters plus a custom filter and a filter reference.
  • And/Or/Not: Logic filters to express logic
  • Exception: Filters based on exception type
  • Payload: Checks the payload type to decide if it should be processed
  • Expression: Checks for expression
  • Message:
  • Custom: Refers a user implemented filter class
  • Message Property
  • Wildcard
  • Schema Validation
  • Regex
  • Idempotent

Configuring Filters

Message Filter

  • Create a new project. Drag a HTTP Connector to source section of the flow and configure it to lsiten on /filters path
  • Drag and drop a Message filter after the HTTP connector
  • Open the properties tab for the filter and click on the plus icon beside the Nested Filter text box
  • Select expression filter from the next screen and click Next. Enter the expression #[message.inboundProperties.'http.query.params'.name!=null]
  • Drag a logger in the end
  • Send the request to localhost:8081/filters. If all good, then it should throw FilterUnacceptedException
